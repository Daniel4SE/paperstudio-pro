# NeurIPS / ICML / ICLR Paper Formatting

## Document Class & Style Package

### NeurIPS
```latex
\documentclass{article}
\usepackage[final]{neurips_2024}   % or [preprint] for non-anonymous arXiv version
% Do NOT pass any options to natbib — neurips_2024 loads it internally
```

### ICML
```latex
\documentclass{article}
\usepackage[accepted]{icml2024}    % or no option for submission (anonymous)
```

### ICLR
```latex
\documentclass{article}
\usepackage{iclr2025_conference,times}
\input{math_commands.tex}          % ICLR ships a math_commands.tex
```

## Page Limits

| Venue   | Body pages | References   | Appendix              |
|---------|------------|--------------|-----------------------|
| NeurIPS | 9          | unlimited    | unlimited (after refs)|
| ICML    | 8          | unlimited    | unlimited             |
| ICLR    | 8 (soft)   | unlimited    | unlimited             |

> The style files enforce page limits automatically via overlength warnings.

## Anonymization (Double-Blind)

All three venues use **double-blind review**. For the submitted version:

```latex
% NeurIPS — pass no author-revealing option:
\usepackage[]{neurips_2024}

% ICML — pass no option (anonymous by default)
\usepackage{icml2024}

% ICLR — iclr2025_conference is anonymous by default
```

- Never include author names, affiliations, acknowledgements, or self-identifying footnotes in the submission version.
- Use `Anonymous` as author placeholder only if the style requires it (ICLR requires `\iclrfinalcopy` for camera-ready).
- Remove or anonymize references to your own prior work ("In our previous work [X]…" → "In [X]…").

## Paper Structure

```
Abstract (150–250 words, single paragraph)
1  Introduction (3–5 paragraphs)
2  Related Work (thematic paragraphs, NOT bullet lists)
3  Background / Preliminaries (if needed)
4  Method / Proposed Approach
   4.1  Problem Formulation
   4.2  Architecture / Algorithm
   4.3  Theoretical Analysis (lemma/proposition/theorem if applicable)
5  Experiments
   5.1  Datasets and Implementation Details
   5.2  Comparison with State-of-the-Art (main results table)
   5.3  Ablation Study
   5.4  Qualitative Analysis / Visualizations
6  Conclusion (single paragraph)
References
Appendix A, B, C … (proofs, additional experiments, implementation details)
```

## Tables

Always use `booktabs` + `resizebox` for wide tables:

```latex
\begin{table}[t]
\caption{Comparison on benchmark X. Bold: best, underline: second best.}
\label{tab:main}
\centering
\resizebox{\linewidth}{!}{%
\begin{tabular}{lccccc}
\toprule
Method & Venue & Metric1$\uparrow$ & Metric2$\uparrow$ & Metric3$\downarrow$ & Params \\
\midrule
Baseline A  & CVPR'23 & 72.3 & 68.1 & 0.432 & 25M \\
Baseline B  & NeurIPS'23 & 74.1 & 70.5 & 0.411 & 31M \\
\midrule
\textbf{Ours} & -- & \textbf{77.8} & \textbf{73.2} & \textbf{0.388} & 28M \\
\bottomrule
\end{tabular}}
\end{table}
```

- Caption goes **above** the table (`\caption` before `\centering`).
- Use `\uparrow`/`\downarrow` to indicate metric direction.
- Use `\textbf{}` for best and `\underline{}` for second best.

## Figures

```latex
\begin{figure}[t]
  \centering
  \includegraphics[width=\linewidth]{figures/fig_architecture.pdf}
  \caption{Overview of \textsc{ModelName}. The encoder (left) processes ...}
  \label{fig:architecture}
\end{figure}
```

- Always use `[t]` (top placement) for single figures.
- Use `\begin{figure*}[t]` for full-width (two-column) figures in ICML (which is two-column style).
- All figures must be PDF (vector), generated by Python scripts in `figures/`.
- File naming: `fig_architecture.pdf`, `fig_ablation.pdf`, `fig_qualitative.pdf`.

## Mathematics

```latex
% Theorem environments (define in preamble)
\newtheorem{theorem}{Theorem}
\newtheorem{proposition}{Proposition}
\newtheorem{lemma}{Lemma}
\newtheorem{corollary}{Corollary}
\newtheorem{definition}{Definition}
\newtheorem{remark}{Remark}

% Use \begin{align} for multi-line derivations
\begin{align}
  \mathcal{L}(\theta) &= \mathbb{E}_{(x,y)\sim\mathcal{D}}\left[
    -\log p_\theta(y \mid x)
  \right] \label{eq:loss} \\
  &= \mathbb{E}\left[ \sum_{t=1}^{T} -\log p_\theta(y_t \mid y_{<t}, x) \right]
\end{align}
```

- Always label equations referenced in text: `\label{eq:...}` and cite with `\eqref{eq:...}`.
- Bold vectors: `\mathbf{x}`, bold matrices: `\mathbf{W}`, calligraphic sets: `\mathcal{D}`.
- Use `\mathbb{R}^{d}` for real vector spaces, `\mathbb{E}[\cdot]` for expectation.

## Citations

NeurIPS/ICML/ICLR all use **natbib** by default:

```latex
% Author-as-subject: Smith et al.~\citep{smith2024} show that...
% Parenthetical:     This approach~\citep{smith2024} was later extended...
\citep{key}      % (Smith et al., 2024)
\citet{key}      % Smith et al. (2024)
\citealt{key}    % Smith et al., 2024  (no parentheses)
```

BibTeX style:
- NeurIPS: uses built-in `\bibliographystyle` — do NOT specify manually.
- ICML: `\bibliographystyle{icml2024}`
- ICLR: `\bibliographystyle{iclr2025_conference}`

## Checklist (NeurIPS Mandatory)

NeurIPS requires a **paper checklist** after the conclusion and before references:

```latex
\begin{enumerate}
\item {\bf Claims} — Do the main claims match theoretical and experimental results? \answerYes{}
\item {\bf Limitations} — Are limitations discussed? \answerYes{}
\item {\bf Theory Assumptions} — Are theoretical assumptions stated? \answerNA{}
% ... (see NeurIPS 2024 checklist template)
\end{enumerate}
```

ICML and ICLR do **not** require this checklist section.

## Common Pitfalls

- ❌ Do NOT use `\paragraph{}` as a subsection substitute — use `\subsection{}`
- ❌ Do NOT use `\begin{itemize}` in the main body (only in the contributions list)
- ❌ Do NOT hardcode font sizes (`\small`, `\footnotesize`) inside figure captions — let the style handle it
- ❌ Do NOT use `\hline` in tables — use `booktabs` rules
- ❌ Do NOT include acknowledgements in the anonymous submission
- ✅ DO use `\linewidth` instead of `\textwidth` inside floats
- ✅ DO run `pdflatex → bibtex → pdflatex → pdflatex` for final compilation
